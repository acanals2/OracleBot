The AI-built problem

AI codegen ships fast. It also ships blind spots.

Lovable, v0, Bolt, Cursor, Replit Agent, Claude Code — they generate working apps in minutes. They also generate apps with the anon Supabase key sitting in main.js, RLS policies missing on every public table, hardcoded API keys in the bundle, and prompt-injection vulnerabilities in every chat endpoint. By default. Every time.

  • [ × ]

    Exposed credentials in client bundles

    Your Supabase anon key, your OpenAI key, your Stripe key. Anyone who opens DevTools can read them. Most AI codegen workflows ship them by default.

  • [ × ]

    Missing Row Level Security

    Supabase's anon key is meant to be public — but only when RLS is on. AI agents scaffold the project, skip the policies, and your users table is world-readable.

  • [ × ]

    Prompt injection in user-content paths

    RAG over user docs. Chat over user input. AI codegen scaffolds the LLM call without the system prompt isolation. One adversarial input drains your tool budget or leaks your prompt.

  • [ × ]

    Production debug surfaces

    Next.js dev overlay reachable on prod. GraphQL introspection on. Prisma Studio at /__prisma. Default error pages leaking stack traces. Tests pass. Lighthouse passes. None of those find this.

Commitments

What OracleBot will never do.

Some constraints are how the product works. These are how the product stays the product.

  • ◆

    Never run against production.

    OracleBot refuses targets that look like live production environments. Every run requires a verified staging or pre-launch target — DNS TXT or OAuth proof — and we monitor for production tells (live TLS chain on apex, real user traffic, public DNS) and abort the run if we see them.

  • ◆

    Never run against a domain you don't own.

    Verification is gating, not a checkbox. No verified ownership, no run. We will not "make an exception" because a deal is on the line — the constraint is the product.

  • ◆

    Never inflate metrics for any platform we don't operate.

    OracleBot is a pre-launch readiness tool. It is not a service for boosting signups, followers, transactions, or any other number on a platform someone else runs. Every run is logged with the verified target and operator identity; abuse is investigable and we cooperate with takedown requests against operators who circumvent our verification.

  • ◆

    Never sell or share what we observe.

    What your synthetic users do in your staging environment stays in your account. We retain run logs (request counts, error rates, flagged flows) for the duration of the run and a configurable window after — and that's it. No training corpora, no benchmarks across customers, no leaks.

These commitments are written into our terms and enforced in the runner. If we ever loosen them, you'll see it in a changelog before it ships.
